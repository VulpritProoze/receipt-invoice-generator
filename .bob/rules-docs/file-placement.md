# Documentation Mode: File Placement

**Purpose**: Defines where different document types go, naming conventions, and directory organization rules.

---

## Directory Structure

```
docs/
├── architecture/      # System design and module docs
├── decisions/         # ADRs (Architecture Decision Records)
├── getting-started/   # Setup and usage guides
├── plans/             # Task plans and roadmaps
├── reports/           # Test, refactor, commit reports, session logs
└── templates/         # Documentation templates
```

---

## Placement Rules

| Doc Type | Directory | doc_id Format | Naming Pattern |
|----------|-----------|---------------|----------------|
| Architecture | `docs/architecture/` | `ARCH-NNN` | `kebab-case.md` |
| Decisions (ADRs) | `docs/decisions/` | `DEC-NNN` | `DEC-NNN-description.md` |
| Plans | `docs/plans/` | `PLAN-NNN` | `kebab-case.md` |
| Test Reports | `docs/reports/test-reports/` | `REP-TEST-NNN` | `REP-TEST-NNN-description.md` |
| Refactor Reports | `docs/reports/refactor-reports/` | `REP-REFACTOR-NNN` | `REP-REFACTOR-NNN-description.md` |
| Commit Reports | `docs/reports/commit-reports/` | `REP-COMMIT-NNN` | `REP-COMMIT-NNN-description.md` |
| Session Logs | `docs/reports/session-logs/unresolved/` | N/A | `SES-[ISO-timestamp].md` |
| Getting Started | `docs/getting-started/` | `GS-NNN` | `kebab-case.md` |
| Templates | `docs/templates/` | N/A | `[type]-template.md` |

**Special rules:**
- **ADRs**: Never delete deprecated ADRs — mark as `deprecated` or `superseded`. Maintain chronological numbering.
- **Test Reports**: Always generated by `/run-tests` — never hand-written.
- **Refactor Reports**: Must list every file modified/created/deleted and explain reasoning.
- **Session Logs**: Auto-created at session end. Never manually edit resolved logs.
- **Templates**: Changes affect all future docs of that type.

---

## File Naming Conventions

**General rules:**
- Always use kebab-case: `my-document.md`
- Be descriptive: `invoice-module.md` not `module.md`
- Use singular nouns: `invoice-module.md` not `invoices-modules.md`
- No spaces or underscores

**Numbered documents:**
- Format: `[PREFIX]-[NNN]-[description].md`
- Number: Three-digit zero-padded (001, 042, 123)
- Find highest existing number, increment by one, zero-pad

---

## Special Files

**AGENTS.md** (repository root):
- Only modify when explicitly requested
- Never delete historical entries
- Maintain phase log table format

**README.md** (repository root):
- Only modify when explicitly requested
- Maintain existing narrative structure
- Update version/date when content changes

---

## Placement Decision Tree

1. **Code change?** → `docs/reports/[refactor|commit|test]-reports/`
2. **Decision?** → `docs/decisions/`
3. **System design?** → `docs/architecture/`
4. **Planning?** → `docs/plans/`
5. **Onboarding/usage?** → `docs/getting-started/`
6. **Template?** → `docs/templates/`
7. **AGENTS.md or README.md?** → Update in place at root
8. **Unclear?** → Stop and ask for clarification

---

## When Placement is Unclear

Stop — do not guess. Output:
```
NEED MORE INFORMATION
Document type unclear. Could fit in [option 1] or [option 2].
Please specify target directory or clarify document type.
```
