### 1. Database Integration and Security Tests (`src/lib/db/*`)

- **Files**: `database.integration.test.ts`, `database.security.test.ts`
- **Error**: Sequences returned unexpected values (e.g. Expected 1, Received 3), and list endpoints returned invoices from other test blocks.
- **Root cause**: Despite adding a direct module mapping for `@/lib/redis` to force usage of `mockRedis`, tests executing sequentially within the same suite or concurrently across files are still leaking data. The singleton `MockRedisClient` might not be reliably cleared by `beforeEach` due to import caching or references across different modules.

- **Root cause**: Same Redis state isolation issues. Data from onboarding workflows leaks across test boundaries causing `getCompanyConfig` to fail deep equality checks or return `undefined`.
