## Hand-off Notes

- The Vercel runtime crash was caused by the static `import * as sqlite...` imports in the `db` adapter files (e.g. `invoices.ts`), which forced `db.sqlite.ts` to evaluate. `db.sqlite.ts` statically ran `new Database(dbPath)` and tried to create `.dev/` at module load time, crashing Vercel functions (read-only filesystem) even when `USE_REDIS=true`.
- Switched all 4 DB adapters to load their SQLite implementation dynamically via `await import()`.
- Rewrote `db.sqlite.ts` to utilize a `Proxy` so the database connection is only opened on the first actual method call, preventing any filesystem access at module evaluation time.
- Next.js 16 startup instrumentation is now added in `src/instrumentation.ts` to log environment variables on startup. Sensitive variables (`UPSTASH_REDIS_REST_TOKEN`, `NEXTAUTH_SECRET`, `ADMIN_PASSWORD`) are masked.
- Removed the deprecated `experimental.instrumentationHook` flag from `next.config.ts` to resolve build warnings, as instrumentation is supported natively in Next.js 16.
